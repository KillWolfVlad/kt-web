- `title` index.html и PWA должен быть `KT Web`
- `description` index.html и PWA должен быть `Unofficial KinoTrend Web Version`
- Сверху должен отображаться "липкий" navbar
- Состав navbar:
  - Левая часть - бренд надпись с названием приложения `KT Web`
  - Правая часть
    - Кнопка с фильтрами (эмодзи 🔍)
      - Если хотя бы один фильтр активен, на кнопке отображается маленький бейдж-кружок цвета `--color-primary` в правом верхнем углу
      - Кнопка с фильтрами отрывает такое-же по внешнему виду как и настройки приложения (см `specs/2026-05-18-3-settings-dialog`) модальное диалоговое окно с фильтрами
        - Инпуты для фильтра фильмов:
          - Фильтр по названию фильма
          - Числовой фильтр по рейтингу фильма (>= заданное значение)
        - Напротив каждого фильтра должна быть кнопка быстрой очистки
        - У самой формы есть две кнопки
          - `Очистить`, которая очищает все фильтры и закрывает форму
          - `Применить`, которая сохраняет фильтры и закрывает форму
        - Фильтры хранятся только в памяти приложения и после перезагрузки сбрасываются
        - Фильтры применяются после закрытия модального диалогового окна
        - Фильтры объединяются через **И**
    - Кнопка которая открывает настройки приложения (см `specs/2026-05-18-3-settings-dialog`)
- Когда любое модальное диалоговое окно (фильтры или настройки) открыто, скролл основного контента должен быть заблокирован (scroll lock)
  - При блокировке ширина скроллбара должна компенсироваться через padding-right на body, чтобы контент не дёргался
- Если контент модального диалогового окна превышает его высоту (или высоту окна браузера), должен появляться внутренний скролл внутри диалога
- Модальное окно с настройками внизу должно отображать:
  - `Git Commit: [первые 6 символов import.meta.env.VITE_GIT_COMMIT_SHA]`
    - Если `import.meta.env.VITE_GIT_COMMIT_SHA` не задан, то используй `<dev>`
  - `Лицензия: Apache License 2.0`
  - `Исходный код: https://github.com/KillWolfVlad/kt-web`
