# 2026-05-18-6-movie-view

- Схему данных смотри в `rules/movies-data-schema.md`
- При нажатии на элемент сетки (см `specs/2026-05-18-4-movies-list`) должно отрыться такое-же по внешнему виду как и настройки приложения (см `specs/2026-05-18-3-settings-dialog`) модальное диалоговое окно с просмотром информации о фильме
  - На компьютере и планшете высота диалога определяется контентом (`height: auto; max-height: 80vh`) в отличие от настроек, где высота фиксированная
  - На мобильных устройствах (<=767px) диалог занимает всю ширину и высоту экрана, как и остальные модальные окна
    - На мобильных устройствах модальное диалоговое окно можно закрыть смахиванием (свайпом) вправо
  - Заголовок модального диалогового окна: `Movie.nameRU`
    - В правой части заголовка расположен крестик для закрытия модального диалогового окна
  - Тело модального диалогового окна:
    - Ссылка на Кинопоиск (ссылка формируется по правилу `https://www.kinopoisk.ru/film/${encodeURIComponent(Movie.filmID)}`)
    - `Movie.country`
    - `Movie.genre`
    - `[Movie.ratingAgeLimits]+`
    - `Movie.filmLength`
    - `Movie.directors`
    - `Movie.actors`
    - `Movie.description`
    - Список `Movie.torrents`
      - `[Torrent.type] / [Torrent.date] / [Torrent.audio] / Seeders: [Torrent.seeders] / Leechers: [Torrent.leechers]` 💾 🔗 🏡
        - 💾 - кнопка которая позволяет скачать `Torrent.link`
        - 🔗 - кнопка которая позволяет скопировать `Torrent.magnet` в буфер обмена (**ВАЖНО**: данный функционал должен работать в Safari на iOS, iPadOS и MacOS)
          - Приоритет: сначала `document.execCommand("copy")` (синхронно в контексте жеста), затем `navigator.clipboard.writeText` как fallback
          - В случае неудачи копирования должно отобразиться красное toast уведомление с текстом ошибки
        - 🏡 - кнопка которая вызывает WebHook (см `specs/2026-05-18-3-settings-dialog`)
          - Во время вызова кнопка должна стать `disabled`, после окончания вызова кнопка должна перестать быть `disabled`
          - После успешного должно отобразиться успешное (зеленое) toast уведомление сверху по центру экрана
          - В случае неудачного вызова должно отобразиться не успешное (красное) toast уведомление сверху по центру экрана с сообщением ошибки
          - Toast уведомление должно закрываться при нажатии на него или через 3 секунды
          - Вызов WebHook должен быть произведен так:
            - `POST [WebHook]`
            - `Content-Type: application/json`
            - `Body`: `{"action": "DOWNLOAD_BY_MAGNET_LINK", "magnetLink": "[Torrent.magnet]"}`
    - Кнопка: `Закрыть`, которая закрывает модальное диалоговое окно
    - Также модальное диалоговое окно должно закрываться при нажатии на кнопку `Esc` (тоже- самое должно происходить в настройках и фильтрах)
  - Когда модальное диалоговое окно открыто, скролл основного контента должен быть заблокирован (scroll lock)
    - При блокировке ширина скроллбара должна компенсироваться через padding-right на body, чтобы контент не дёргался
- Контент внутри модального диалогового окна должен быть скроллируемым, если он превышает его высоту или высоту окна браузера (например при открытых DevTools)
- Модальные диалоговые окна настроек и фильтров при нажатии `Enter` должны сохранять текущие результаты и закрываться
