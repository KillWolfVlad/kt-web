# 2026-05-18-3-settings-dialog

- Когда пользователь в первый раз открывает приложение, выведи модальное диалоговое окно с настройками
- В настройках должно быть два инпута:
  - Источник с данными для загрузки списка фильмов (`data.json`) - обязателен для заполнения
    - При первом визите без его заполнения нельзя закрыть модальное диалоговое окно нажатием на крестик или вне окна
    - Кнопка `Сохранить` неактивна, если поле источника данных пустое
  - Путь до WebHook - опционален
- Данные сохраняются при нажатии на кнопку `Сохранить` в `localStorage` (и загружаются в последствии тоже из `localStorage`)
  - После сохранения модальное диалоговое окно закрывается
- Внизу диалога расположена информация (Git Commit, лицензия, исходный код), а под ней — кнопка `Сохранить`
- На декстопе модальное диалоговое окно расположено по центру и занимает 50% экрана по ширине и высоте.
  - На планшете модальное диалоговое окно занимает 75%
  - На мобильных устройствах модальное диалоговое окно занимает всю ширину и высоту
  - На мобильных устройствах модальное диалоговое окно можно закрыть смахиванием (свайпом) вправо
- Фон за модальным диалоговым окном должен быть черным с прозрачностью 80%
- Когда модальное диалоговое окно открыто, скролл основного контента должен быть заблокирован (scroll lock)
  - При блокировке ширина скроллбара должна компенсироваться через padding-right на body, чтобы контент не дёргался
- Если контент модального диалогового окна превышает его высоту (или высоту окна браузера, например при открытых DevTools), должен появляться внутренний скролл внутри диалога
